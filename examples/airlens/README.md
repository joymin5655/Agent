# AirLens Example

This directory preserves the original AirLens agent/runtime mirror as an example of a mature project-specific harness.

It is intentionally not installed by default. Use it as a reference for adapting the generic `core/` policies to a real product domain.

Domain-specific examples here include:

- air-quality data and model policies
- DQSS, AOD, SDID, and Globe specialists
- AirLens Codex skills
- AirLens workflow and GitHub Actions mirrors
- historical agent-harness documentation

When adapting these files to another project, remove product names, concrete repository paths, vendor-specific resources, and local-only settings before promotion to `core/`.
